# AZ-400 study path

Use this path to practice feedback-driven platform delivery: source control, CI/CD, infrastructure as code, secure authentication, observability and continuous operations.

## Source and collaboration

- Explain the main/template/generated repository responsibilities.
- Review CODEOWNERS, Dependabot, version pinning and breaking golden-path version policy.
- Propose branch protection and required checks for path/controller changes.
- Write a pull-request risk note for a lifecycle deletion change.

Evidence: protected-branch/check policy screenshot or documented design and a reviewed sample pull request.

## CI quality gates

Map each check to a defect class:

| Check | Defect class |
| --- | --- |
| Terraform fmt/validate/test | Syntax, contract and module composition |
| TFLint/Checkov/Trivy/Conftest | Quality, security, image/IaC and platform policy |
| Gitleaks | Committed credential/material exposure |
| actionlint/ShellCheck/Hadolint | Workflow, script and container build defects |
| npm/Docker/Helm/kubeconform | Application and delivery artifact correctness |
| Markdown/link/terraform-docs | Operability and documentation drift |

Explain why PR checks should not request Azure OIDC or GitHub App credentials when source validation is sufficient.

## Secure continuous delivery

Trace the exact generated repository/environment OIDC subject, job `id-token` permission, Azure role scope, `PLATFORM_READY` gate and initial dispatch. Compare OIDC with a stored service-principal secret and explain residual risks such as broad subject, broad role, unpinned actions and compromised runner/dependency.

For Container App/AKS, capture immutable image reference and ACR repository ABAC. For AKS, capture required reviewer and Helm output.

## Infrastructure and policy as code

- Explain bootstrap versus shared platform versus workload states.
- Run a plan through Conftest and interpret a rejection.
- Demonstrate Terraform contract tests with mocked providers.
- Explain why a breaking path change publishes v2.
- Review state locking, lease, ETag and fencing for lifecycle operations.

## Observability and feedback

Build queries/dashboards for request latency, deployment result, phase duration, retries, endpoint health, expiry and heartbeat. Design one deduplicated incident workflow and write a concise runbook link/alert payload.

## Release and failure exercise

Choose one failure: OIDC propagation, Terraform partial apply, GitHub 429, Azure residual, transferred repository or lost lease. Inject it safely, show the expected phase/retry/fail-closed action, recover through reconciliation and preserve sanitized evidence.

Deliverable: a pipeline walkthrough and incident review demonstrating that Azure absence precedes repository deletion.
